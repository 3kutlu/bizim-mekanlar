# v2.7.0 — Ayarlar ve Hesap Kontrolleri

Bu sürüm genel Ayarlar merkezini, kullanıcı bazlı sessize alma/engelleme işlemlerini ve hesap dondurma/silme yaşam döngüsünü ekler.

## Kapsam

- Ayarlar merkezi
  - Profilimi düzenle
  - Hesap gizliliği
  - Engellenen hesaplar
  - Sessize alınan hesaplar
  - Bildirim ayarları
  - Çıkış yap
  - Hesabı dondur
  - Hesabımı sil
- Takipçi listesinden kullanıcı çıkarma
- Kullanıcı engelleme ve engeli kaldırma
- Takip edilen kullanıcıyı sessize alma
  - `FOLLOWING_NOTE`
  - `NOTE_REACTION_UP`
  - `NOTE_REACTION_DOWN`
- Ortak koleksiyon bildirimi sessize almadan etkilenmez
- Süresiz hesap dondurma ve girişte yeniden açma
- Kullanıcı adı onaylı, 7 günlük bekleme süreli hesap silme
- Süre sonunda Storage, uygulama verileri ve Supabase Auth hesabının temizlenmesi

## Dağıtım sırası

Aşağıdaki sıra önemlidir. Edge Function kodları yeni tablo ve kolonları kullandığı için migration önce uygulanmalıdır.

### 1. Migration

Supabase SQL Editor üzerinden veya CLI ile çalıştır:

```bash
supabase db push
```

Uygulanacak migration:

```text
supabase/migrations/20260712_v2_7_0_account_controls.sql
```

### 2. Edge Function secret'ı

Kalıcı silme işini çağıracak zamanlayıcı için güçlü ve rastgele bir secret tanımla:

```bash
supabase secrets set ACCOUNT_PURGE_INTERNAL_SECRET="UZUN_RASTGELE_BIR_DEGER"
```

Bu değeri frontend ortam değişkenlerine ekleme.

### 3. Edge Function deploy

```bash
supabase functions deploy request-auth-otp --no-verify-jwt
supabase functions deploy send-web-push --no-verify-jwt
supabase functions deploy manage-account
supabase functions deploy purge-deleted-accounts --no-verify-jwt
```

`manage-account` kullanıcı oturumu gerektirir. `purge-deleted-accounts` ise JWT yerine yalnızca `x-internal-secret` başlığını kabul eder.

### 4. Kalıcı silme zamanlayıcısı

`purge-deleted-accounts` fonksiyonunu en az günde bir, tercihen saatte bir POST isteğiyle çağıran bir zamanlayıcı kur.

İstek biçimi:

```bash
curl -X POST \
  "https://<PROJECT_REF>.supabase.co/functions/v1/purge-deleted-accounts" \
  -H "x-internal-secret: UZUN_RASTGELE_BIR_DEGER"
```

Yanıt, işlenen her kullanıcı için `deleted` veya `failed` durumunu döndürür. Başarısız kullanıcılar sonraki çalıştırmada yeniden denenebilir.

### 5. Frontend deploy

```bash
npm ci
npm run check
vercel --prod
```

## Hesap durumları

| Durum | Davranış |
| --- | --- |
| `ACTIVE` | Normal kullanım |
| `FROZEN` | Süresiz dondurulmuş; girişte yeniden açma ekranı |
| `DELETION_PENDING` | 7 günlük geri dönüş süresi; girişte iptal etme seçeneği |
| `DELETED` | Kişisel alanları anonimleştirilmiş, Auth bağlantısı kaldırılmış hesap |

## Test kontrol listesi

1. Ayarlar dişlisi genel Ayarlar ekranını açıyor.
2. Profil düzenleme mevcut alanları kaydediyor.
3. Gizlilik değişikliği profil özetine yansıyor.
4. Takip edilen kullanıcı sessize alınabiliyor ve ses yeniden açılabiliyor.
5. Sessize alınan kullanıcı için not ve tepki push bildirimleri gelmiyor.
6. Ortak koleksiyon bildirimi gelmeye devam ediyor.
7. Takipçi listesinden çıkarılan kullanıcı artık takipçi görünmüyor.
8. Engellenen kullanıcı arama, akış, profil, bildirim ve bağlantı listelerinde görünmüyor.
9. Engel kaldırıldığında eski takip ilişkisi otomatik dönmüyor.
10. Hesap dondurulduktan sonra oturum kapanıyor.
11. Dondurulmuş hesap yeniden girişte kurtarma ekranına geliyor.
12. Hesabı yeniden açma eski verileri koruyor.
13. Yanlış kullanıcı adıyla silme butonu aktif olmuyor.
14. Doğru kullanıcı adıyla hesap `DELETION_PENDING` oluyor.
15. Yedi gün dolmadan giriş yapıldığında silme iptal edilebiliyor.
16. Yedi gün dolduktan sonra yeniden açma reddediliyor.
17. Purge fonksiyonu fotoğrafları, Auth hesabını ve kullanıcı verilerini temizliyor.
18. Silme tamamlandıktan sonra aynı e-posta ile sıfırdan kayıt yapılabiliyor.

## Geri dönüş

Frontend için önceki Git tag'ine dönülebilir. Migration veri eklediği için yalnızca frontend rollback yapılırsa yeni kolon ve tabloların veritabanında kalması sorun oluşturmaz. Tabloları veya kolonları aceleyle silme; önce yeni fonksiyonların kullanılmadığını doğrula.
