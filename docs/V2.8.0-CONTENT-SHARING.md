# v2.8.0 — Uygulama İçi Paylaşımlar

## Kapsam

- Mekan, not, koleksiyon ve profil paylaşımı
- Paylaşırken uygulama içi gönderme, dış paylaşım veya bağlantı kopyalama seçenekleri
- Yalnızca takip edilen aktif kullanıcılara gönderim
- İsteğe bağlı 100 karakterlik kısa not
- Gelişmeler ekranında Paylaşımlar sekmesi
- Okunmamış paylaşım rozeti ve Realtime yenileme
- Engellenen hesaplar için iki yönlü koruma
- İçeriğin alıcı tarafından görüntülenebilirlik kontrolü
- Saatlik ve kullanıcı bazlı gönderim limiti
- Web Push bildirimi
- Hesap kalıcı silinirken paylaşım kayıtlarının pasifleştirilmesi

## Veritabanı

Önce kontrol edin:

```cmd
npx.cmd supabase db push --dry-run
```

Yalnızca aşağıdaki yeni migration görünmelidir:

```text
20260712010000_v2_8_0_content_shares.sql
```

Ardından:

```cmd
npx.cmd supabase db push
```

## Edge Function

```cmd
npx.cmd supabase functions deploy send-content-share-push
```

Fonksiyon mevcut Web Push VAPID secret'larını kullanır:

- `WEB_PUSH_VAPID_PUBLIC_KEY`
- `WEB_PUSH_VAPID_PRIVATE_KEY`
- `WEB_PUSH_VAPID_SUBJECT`

## Frontend

```cmd
npm ci
npm run check
vercel --prod
```

## Temel test

1. A kullanıcısı B kullanıcısını takip etsin.
2. A bir mekan detayında paylaş butonuna bassın.
3. “Bizim Mekanlar'da gönder” ile B'yi seçsin.
4. B hesabında Gelişmeler > Paylaşımlar sekmesinde kayıt görünmeli.
5. Karta basınca ilgili mekan uygulama içinde açılmalı.
6. Aynı test not, koleksiyon ve profil için tekrarlanmalı.
7. A veya B diğerini engellediğinde yeni gönderim reddedilmeli.
8. Aynı içerik aynı kullanıcıya 5 dakika içinde tekrar gönderilememeli.
